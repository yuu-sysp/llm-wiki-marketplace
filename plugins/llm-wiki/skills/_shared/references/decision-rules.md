# LLM Wiki 判断基準

## どのフォルダに置くか

- 一般化できる技術知見（API・パターン・ハマり）→ `concepts/`
- 「なぜこの選択をしたか」の設計判断（選択肢・理由・トレードオフ）→ `notes/`
- 特定の画面・人・プロジェクトの説明 → `pages/`
- 複数ソース横断の比較・統合 → `type: synthesis`（置き場所は concepts/ か notes/）

## 新規ページを作る条件（3つとも満たすとき）

1. 既存ページに独立した扱いが無い
2. 1セクションに収まらない深さがある
3. 複数ページから参照される（再利用性がある）
→ どれか欠けるなら「既存へ追記」か「作らない」を選ぶ。

## 既存ページを更新するとき

- 関連が直接あり、既存記述を補強・訂正・反証する情報のとき
- `updated` を今日に更新し、`source` を追記
- 矛盾は消さず `## 異論・補足` に両論併記する

## ページ分割の条件（3つとも）

1. 1セクションが単独で ~500〜1000字を超える
2. その話題が独立して外部から参照される
3. 分割で理解しやすくなる
→ 抽出して新ページ化、元は要約＋リンクに置換、`related` と log を更新。

## 取り込み・保存しない基準

- 既存知識の単なる組合せ／具体的すぎて再利用性が低い／個人的な決定記録 → 保存しない
- 迷ったら「保存しない」を既定に。必要になったら後で ingest する。
